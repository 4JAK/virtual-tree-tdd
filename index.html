Code more efficiently with TDD

  • Clear Focus

    You're more productive while coding, and TDD helps keep that productivity high by narrowing your focus. It forces you to think about smaller chunks of functionality at a time rather than the application as a whole.

  • Tidier Code

    Your tests are only interfacing with public methods, so you have a much better idea of what can be made private. You code more efficent and it reduces the chance of creating un-needed or redundant code.

  • Fewer Bugs

    You save time down the line that would be spent fixing bugs that have popped up and need time to figure out. This also helps define what the bug actually is, as you always need reproducible steps.

  • Living Documentation

    Tests can serve as documentation to a developer. Also you can see the expected inputs a method requires and what you can expect as outcome, all based on the assertions made in the test.

TDD is becoming the standard practice for coders to avoid the obstacles of traditional programming.

Using a clear way to create clean code that finds the bugs and avoids pitfalls that plagued coders in the past it helps build your program in a safer more efficient way
